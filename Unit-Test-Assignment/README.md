# Unit Testing Assignment

## Overview

This repository contains the implementation of different types of software testing techniques including:

* Unit Testing
* Integration Testing
* End-to-End Testing

These tests are implemented using widely used testing frameworks such as **JUnit, TestNG, and Selenium**.

The purpose of this assignment is to understand how different testing strategies are used to verify software functionality at different levels of an application.

---

## Testing Types Implemented

### 1. Unit Testing

Unit testing focuses on testing **individual functions or methods** of an application.

Framework used:

* **JUnit**

Example:

* Testing calculator operations such as addition, subtraction, multiplication, and division.

---

### 2. Integration Testing

Integration testing verifies that **multiple modules or components of an application work correctly together**.

Framework used:

* **TestNG**

Example:

* Testing combined operations such as performing addition followed by multiplication.

---

### 3. End-to-End Testing

End-to-end testing simulates **real user interactions with the system** and verifies the entire workflow.

Framework used:

* **Selenium WebDriver**

Example:

* Opening a browser
* Navigating to a website
* Performing a search operation

---

## Technologies Used

* Java
* JUnit
* TestNG
* Selenium WebDriver
* GitHub

---

## Learning Outcomes

Through this assignment the following concepts were practiced:

* Understanding different levels of testing
* Writing unit tests using JUnit
* Performing integration testing with TestNG
* Automating browser testing using Selenium
* Improving software reliability using automated tests
